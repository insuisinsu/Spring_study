

 Transaction	: 작업(일)을 처리하는 최소 단위
 	
 	- All or Nothing  // 전부 되거나 안되면 다 안되거나
 
 	- Insert, Update, Delete 구문에서 트랜잭션이 발생
 	- Oracle DB 는 Insert, Update, Delete 구문은
 		앞에 begin Transaction 이 자동으로 등록됨
 		but, commit 은 명시해줘야 함
 	- MSSQL, MySQL 은 bdgin Transaction, commit 이 자동으로 등록됨
 	- Connection 객체는 기본셋이 자동 commit;
 	- commit; 전 까지는 메모리에만 적용, commit; 실행 후에 DB에 저장
 	- 모든 정보가 log 에 기록됨 (트랜잭션 로그)
 	- 백업된 시점이 아닌, 오류가 발생된 시점으로 복구
 	- 모든 Insert, Update, Delete 구문 내에서 트랜잭션 처리를 해야함
 	
 	- Spring 의 트랜잭션 처리는 AOP 를 사용해서 처리함
 		중복된 코드를 방지할 수 있음
 		XML 설정만 가능 / 어노테이션 사용불가
 	- Spring 에서 트랜잭션 처리를 하기 위해서는 트랜잭션 관리자를 생성해야 함
 	- Spring 에서 트랜잭션 처리는 메소드 단위로 처리됨
 	
 	ex) 
 		A 통장에서 B 통장으로 1억을 이체
 		1) update A - 1억
 		2) update B + 1억
 		-> 하나의 트랜잭션으로 작동해야 함
 			? 트랜잭션이 2개라면 하나만 실행되면 큰일남
